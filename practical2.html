Deadlock & Banker's Algorithm

Configure allocation, maximum need matrices and available resources. The algorithm checks system safety and generates a safe execution sequence if one exists.

📚 Theory

Banker's Algorithm (Dijkstra, 1965) is a deadlock avoidance algorithm. It checks if granting a resource request leaves the system in a safe state.

  • Allocation Matrix — Resources currently held by each process
  • Maximum Matrix — Maximum resources a process may need
  • Need Matrix = Maximum − Allocation
  • Available Vector — Resources not allocated to any process
  • System is safe if there exists an ordering (safe sequence) where all processes can finish
